# QA API Testing Project – Postman CRUD

## 📌 Descripción
Este proyecto simula pruebas de QA sobre una API REST utilizando Postman.  
Se realizaron validaciones de endpoints CRUD (Create, Read, Update, Delete) aplicando pruebas funcionales y verificación de respuestas.

El objetivo es demostrar conocimientos prácticos de testing de APIs y diseño de casos de prueba.

---

## 🧪 Herramientas utilizadas
- Postman
- API pública: https://jsonplaceholder.typicode.com
- JavaScript (Postman Tests)

---

## 🔄 Funcionalidades testeadas

### GET Users
- Validación de status code 200
- Validación de respuesta en formato array

### GET User by ID
- Validación de status code 200
- Verificación de estructura de respuesta (id, name, email)

### POST Create User
- Validación de status code 201
- Validación de datos enviados

### PUT Update User
- Validación de status code 200
- Verificación de datos actualizados

### DELETE User
- Validación de status code 200 o 204

---

## 📋 Test Cases
Se incluyen casos de prueba diseñados para escenarios positivos y negativos, con pasos y resultados esperados documentados en formato estructurado.

---

## 🚀 Objetivo del proyecto
Simular un flujo real de QA en un entorno de desarrollo, aplicando testing funcional sobre APIs REST y validaciones automáticas básicas.
